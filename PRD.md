# PRD — GitHub 热门解读

## 一句话定位

每天 AI 解读热门开源项目，发现好工具。

---

## 目标用户

自用，专注 vibe coding 的开发者，需要快速发现 AI 辅助编程相关工具。

---

## 系统架构

```
每天北京时间 09:00 → GitHub Actions 触发
  ↓
scripts/fetch-and-analyze.ts
  1. GitHub Search API 抓各榜单
  2. 与 projects.json 比对，找新项目
  3. 新项目：抓 README/Issues → 调 Gemini → 存 projects.json
  4. 所有项目：更新 stats.json 数值
  5. 计算 todayStarsDelta（今日stars - 昨日daily快照stars）
  6. 生成 data/daily/YYYY-MM-DD.json
  7. 选今日亮点 → git push
  ↓
Vercel 检测到新 commit → 自动部署
  ↓
用户打开网页 → 读 JSON → 秒显示
```

## V1 功能范围

### 5个Tab

| Tab | 说明 | 数据来源 |
|-----|------|---------|
| 📈 今日热门 | 今日创建，Star最多 | GitHub Search（created:>昨天，stars desc） |
| 📅 本周热门 | 7天内创建，Star最多 | GitHub Search（created:>7天前，stars desc） |
| 🌟 本周新星 | 3月内创建，增速快 | GitHub Search（created:>3月前，stars>100） |
| 🤖 Vibe Coding | AI辅助编程相关项目 | projects.json 中 isVibeCoding=true |
| 🏆 经典项目 | 总Star最多的项目 | projects.json 中 stars>10000 |
| ❤️ 我的收藏 | 用户收藏 | 浏览器 localStorage |

> 今日/本周热门是"最近创建且Star最多"的近似方案，非真实增量排序（V2升级）

### 今日亮点

- 页面顶部置顶展示 2~3 个最值得关注的项目
- 优先推荐 Vibe Coding 适配度高的
- 附带一句话推荐理由

### 项目列表

- 每页20条，点击「加载更多」追加
- 每行显示：项目名↗、Star数、今日增量、Fork数、Issue数、赞助者数、语言、License、最近提交时间、Vibe标签、❤️收藏按钮、一句话简介
- 点击项目名 → 新标签页打开 GitHub
- 点击行其他区域 → 右侧展开 AI 解读

### 搜索和筛选

- 搜索：关键词过滤（项目名/描述/AI解读内容）
- 分类筛选（14类）：LLM框架 / AI Agent / AI编程助手 / CLI工具 / 前端框架 / 后端框架 / 数据库 / DevOps / 测试工具 / 安全 / 文档工具 / 移动开发 / Prompt工程 / 其他
- 排序：Stars / 今日增量 / 创建时间 / 最近提交 / Vibe评分

### AI 解读面板

- **桌面端**：固定右侧栏（约40%宽度）
- **移动端**：全屏覆盖，顶部有关闭按钮
- 内容预生成，点击秒显示，无需等待
- 右上角复制图标（默认纯文本，下拉可选 Markdown）
- 🔄 “重新生成”按钮：AI 解读不满意时一键重新调用 Gemini，覆盖旧解读

| 字段 | 说明 |
|------|------|
| 🎯 一句话定位 | 30字以内 |
| 🎬 使用场景 | 3个"你做X→它帮你做Y" |
| 👤 适合人群 | 一句话具体场景 |
| 🚀 快速上手 | 2~3步最简命令 |
| 🆚 竞品对比 | 2~3个竞品表格 |
| 💰 是否免费 | 免费/部分免费/付费 |
| 🤖 Vibe适配度 | 1~5星 + 理由 |
| 💬 社区活跃度 | 基于Issue开闭比 |

### 收藏功能

- 每个项目行右侧 ❤️ 按钮，点击切换，已收藏显示红色
- 存 localStorage，无需后端
- 收藏上限200个

### 数据更新提示

- 顶部显示「数据更新于 XX月XX日 09:00」
- 超过25小时未更新显示黄色警告

### 运行状态角标

- 页面右下角常驻小角标
- 绿色 ⚫ = 上次运行成功
- 黄色 ⚫ = 上次运行有部分失败（跳过了某些项目）
- 红色 ⚫ = 上次运行 fatal 错误
- 悬停显示 tooltip：更新时间 + 错误条数

### 自动错误通知

- 脚本运行出现 fatal 错误，或超过5个项目失败时
- 自动在 GitHub 仓库创建 Issue，包含错误详情和处理建议
- 不需要手动去 Actions 日志里找问题

- Next.js 14 + App Router（静态SSG）
- Tailwind CSS，极简暗色风格
- @octokit/rest + GitHub Search API

- GitHub Actions cron + Vercel 自动部署

## 视觉设计

- **项目名称**：GitHub 热门解读
- **Slogan**：每天 AI 解读热门开源项目，发现好工具
- **Header**：标题 + Slogan + 更新时间 + 右上角暗/浅色切换按钮
- **配色方案（暗色模式）**：
  - 背景：深蓝 `#0f172a`（参考 Tailwind 暗色）
  - 文字：低对比浅灰 `#94a3b8`，关键数字高亮白 `#f1f5f9`
  - Vibe Coding 标签：紫→蓝渐变（`#8b5cf6` → `#3b82f6`），AI 科技感
  - 今日亮点卡片：金色 `#f59e0b`
- **暗/浅色切换**：Header 右上角按钮，默认暗色
- **过渡效果**：极简无动画，切换 Tab / 筛选时列表直接替换
- **复制功能**：解读面板右上角复制图标，默认纯文本，下拉可选 Markdown

## 移动端设计

**设计原则**：碎片时间快速浏览，精简显示、一指操作。

| 区域 | 交互方式 |
|------|---------|
| **项目列表** | 精简显示 4 项：Star + 增量 + 语言 + Vibe 标签，其余展开后显示 |
| **AI 解读** | 点击项目行 → 手风琴式就地展开解读，再点收起 |
| **项目名称** | 点击名称 → 新标签页打开 GitHub（不触发展开） |
| **搜索** | 搜索框固定顶部，始终可见 |
| **筛选/排序** | 点击 🔽 漏斗图标 → 弹出筛选面板（分类 + 排序选项） |
| **今日亮点** | 横向滑动卡片 |
| **Tab 切换** | 横向标签栏，可滑动 |
| **完整指标** | 展开解读时同时显示 Fork/Issue/赞助者/提交时间等 |

## 定时任务补充

- **Keep-alive**：每月 30 号自动发一个空 commit，防止 GitHub Actions 因 60 天无活动被暂停
- **API Key 读取策略**：优先读环境变量 → 文件不存在时从 `Github Token.txt` / `Googole Ai Studo Api.txt` 读取 → 都找不到则打印友好提示退出
